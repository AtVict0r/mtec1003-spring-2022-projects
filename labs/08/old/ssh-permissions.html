SSH - some background

  • stands for Secure Shell
  • protocol for connecting securely to a remote computer by establishing an encrypted connection

Why use SSH?

  • security
    • connection is secure
    • nothing is sent in clear text

How does it work?

  • Uses public key cryptography
  • You have a public key and a private key
  • Public key encrypts the data and private key decrypts it
  • It is safe to give out your public key but NOT private!

What are the permissions of this file? →

-rw-r--r--  1 bree  staff  0 Jan 14 16:51 file_name
  • File is owned by the User "bree"
  • User (bree) can read and write
  • File is owned by the Group "staff"
  • Group (staff) can read.
  • World (everyone else) can read.

chmod

change permissions

chmod [user, group or world] [plus or minus] [read, write or execute] [filename.sh]
	# allow user to execute
	chmod u+x filename.sh
	# prevent world from writing
	chmod o-w filename.txt

or, you can use a numeric notation:

chmod 600 filename
  • 600 means: rw------- owner may read and write; all others have no rights
  • 700 means: rwx------ owner may read, write, and execute; all others have no rights

chown

change owner and group

chown bree:admins filename.txt

change owner

chown bree filename.txt

change group

chown :admins filename.txt
